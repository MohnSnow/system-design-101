# 🔄 Git工作流详解

> 理解Git的工作流，才能真正用好它

![Git工作流](./git-workflow.png)

Git是分布式版本控制系统，允许多个开发者同时在同一项目上工作 👇

Git的工作流涉及几个关键区域：
📌 工作区（Working Directory）— 你编辑文件的地方
📌 暂存区（Staging Area）— git add后文件进入这里
📌 本地仓库（Local Repository）— git commit后保存到这里
📌 远程仓库（Remote Repository）— git push后同步到这里

💡 理解这4个区域之间的数据流转，是掌握Git的关键。
