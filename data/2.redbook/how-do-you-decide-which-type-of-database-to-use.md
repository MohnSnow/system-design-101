# 💾 如何选择数据库类型

> 数据库那么多，到底该用哪个？

![数据库选型](./how-do-you-decide-which-type-of-database-to-use.png)

数据库有上千种，选型指南 👇

📌 **关系型数据库** — 几乎什么都能解决（MySQL、PostgreSQL）

📌 **内存存储** — 速度快但数据量有限，适合快速操作（Redis）

📌 **时序数据库** — 存储和管理带时间戳的数据（InfluxDB、TimescaleDB）

📌 **图数据库** — 适合非结构化对象间的复杂关系（Neo4j）

📌 **文档存储** — 适合大量不可变数据（MongoDB）

📌 **宽列存储** — 大数据、分析、报表等需要反范式化数据的场景（Cassandra、HBase）

💡 选数据库的原则：先用关系型数据库，遇到瓶颈再考虑专用数据库。不要为了用新技术而用新技术。
