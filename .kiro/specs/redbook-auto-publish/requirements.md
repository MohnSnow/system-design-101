# 需求文档：小红书自动发布工具

## 简介

构建一个基于 Python + Playwright 的小红书自动发布工具，通过浏览器自动化操作小红书创作者平台网页版，将本地 Markdown + 图片文件对自动发布到小红书草稿箱。工具分为两个阶段：单篇发布和批量发布（每次5篇）。

## 术语表

- **发布工具（Publisher）**：本项目的 Python 脚本，负责解析本地文件、驱动浏览器自动化操作，将内容发布到小红书草稿箱
- **内容组（Content_Pair）**：一个 Markdown 文件与同名图片文件（png/jpg/jpeg）的配对，构成一篇待发布的小红书笔记
- **待发布目录（Pending_Directory）**：`data/1.redbook/` 目录，存放待发布的内容组
- **已发布目录（Published_Directory）**：`data/0.redbook_published/` 目录，存放已成功发布的内容组
- **创作者平台（Creator_Platform）**：小红书创作者中心网页版（https://creator.xiaohongshu.com）
- **草稿箱（Draft_Box）**：小红书创作者平台中保存未正式发布笔记的区域
- **话题标签（Topic_Tag）**：Markdown 文件末尾以 `#标签名` 格式定义的话题关键词，对应小红书平台的话题功能
- **Cookie 认证（Cookie_Auth）**：通过用户提供的浏览器 Cookie 字符串完成小红书平台的登录认证

## 需求

### 需求 1：内容组解析

**用户故事：** 作为内容运营人员，我希望工具能自动解析待发布目录中的 Markdown 和图片文件对，以便准确识别每一组待发布内容。

#### 验收标准

1. WHEN 发布工具扫描待发布目录时，THE 发布工具 SHALL 识别所有 Markdown 文件，并按文件名排序生成内容组列表
2. WHEN 发布工具解析一个 Markdown 文件时，THE 发布工具 SHALL 在同目录下查找同名的图片文件（扩展名为 .png、.jpg 或 .jpeg）
3. IF 某个 Markdown 文件没有对应的同名图片文件，THEN THE 发布工具 SHALL 跳过该内容组，并在控制台输出提示信息，告知用户需手动处理
4. WHEN 发布工具解析 Markdown 文件时，THE 发布工具 SHALL 从第一个 `# ` 开头的行提取标题文本（去除 `# ` 前缀）
5. WHEN 提取的标题超过20个字符时，THE 发布工具 SHALL 自动截断标题至20个字符以内
6. WHEN 发布工具解析 Markdown 文件时，THE 发布工具 SHALL 提取标题行之后的所有内容作为正文（去除标题行本身）
7. WHEN 发布工具解析 Markdown 文件时，THE 发布工具 SHALL 从文件末尾的 `#标签名` 格式文本中提取所有话题标签

### 需求 2：Cookie 认证

**用户故事：** 作为内容运营人员，我希望通过提供浏览器 Cookie 来完成小红书平台认证，以便工具能代替我操作创作者平台。

#### 验收标准

1. THE 发布工具 SHALL 支持从本地配置文件或环境变量中读取小红书 Cookie 字符串
2. WHEN 启动浏览器自动化会话时，THE 发布工具 SHALL 将 Cookie 注入到浏览器上下文中，完成创作者平台的登录认证
3. IF Cookie 无效或已过期导致认证失败，THEN THE 发布工具 SHALL 终止执行并输出明确的错误提示信息

### 需求 3：单篇发布到草稿箱

**用户故事：** 作为内容运营人员，我希望工具能自动将一组图文内容发布到小红书草稿箱，以便我后续在平台上审核并正式发布。

#### 验收标准

1. WHEN 发布工具执行单篇发布时，THE 发布工具 SHALL 打开创作者平台的发布页面
2. WHEN 发布工具操作发布页面时，THE 发布工具 SHALL 上传内容组中的图片文件
3. WHEN 发布工具操作发布页面时，THE 发布工具 SHALL 填写标题字段（使用解析后的标题）
4. WHEN 发布工具操作发布页面时，THE 发布工具 SHALL 填写正文字段（使用解析后的正文内容）
5. WHEN 内容组包含话题标签时，THE 发布工具 SHALL 尝试在发布页面的话题输入框中搜索并选择匹配的话题
6. IF 话题标签在平台上无法自动匹配，THEN THE 发布工具 SHALL 在控制台输出提示信息，告知用户需手动添加该话题
7. WHEN 所有字段填写完成后，THE 发布工具 SHALL 点击"存草稿"按钮将内容保存到草稿箱
8. IF 发布过程中出现页面加载超时或元素定位失败，THEN THE 发布工具 SHALL 记录错误信息并跳过当前内容组，继续处理下一组

### 需求 4：发布后文件归档

**用户故事：** 作为内容运营人员，我希望已成功发布的文件自动归档到已发布目录，以便清晰区分待发布和已发布的内容。

#### 验收标准

1. WHEN 一组内容成功保存到草稿箱后，THE 发布工具 SHALL 将该内容组的 Markdown 文件和图片文件从待发布目录移动到已发布目录
2. IF 已发布目录不存在，THEN THE 发布工具 SHALL 自动创建该目录
3. IF 文件移动过程中发生错误，THEN THE 发布工具 SHALL 记录错误信息，但保留原文件不删除

### 需求 5：批量发布模式

**用户故事：** 作为内容运营人员，我希望工具支持批量模式，每次自动发布5篇内容到草稿箱，以便提高发布效率。

#### 验收标准

1. WHEN 发布工具以批量模式运行时，THE 发布工具 SHALL 从待发布目录中按文件名排序选取前5组有效内容组
2. WHEN 执行批量发布时，THE 发布工具 SHALL 依次对每组内容执行单篇发布流程（需求3的完整流程）
3. WHEN 批量发布中某一组发布失败时，THE 发布工具 SHALL 记录失败信息并继续处理剩余内容组
4. WHEN 批量发布完成后，THE 发布工具 SHALL 输出发布结果摘要，包含成功数量、失败数量和跳过数量
5. IF 待发布目录中有效内容组不足5组，THEN THE 发布工具 SHALL 发布所有可用的有效内容组，并在摘要中说明实际发布数量

### 需求 6：运行日志与用户反馈

**用户故事：** 作为内容运营人员，我希望工具在运行过程中提供清晰的进度和状态信息，以便我了解发布进展和排查问题。

#### 验收标准

1. WHILE 发布工具运行期间，THE 发布工具 SHALL 在控制台输出当前处理进度（例如"正在发布第 2/5 篇：标题名"）
2. WHEN 每组内容发布完成时，THE 发布工具 SHALL 输出该组的发布结果（成功/失败/跳过）
3. IF 发布过程中出现异常，THEN THE 发布工具 SHALL 输出包含错误类型和上下文信息的日志，便于问题排查
